# Changelog 2025-11

- Implemented copper-pour solver pipeline, enabling automatic copper-pour generation ([copper-pour-solver #2](https://github.com/tscircuit/copper-pour-solver/pull/2))
- Gerber export now respects `covered_with_solder_mask` on copper pours ([circuit-json-to-gerber #62](https://github.com/tscircuit/circuit-json-to-gerber/pull/62))
- `resolvedPcbStyle` is now forwarded to footprint loaders for more accurate KiCad footprint rendering ([core #1617](https://github.com/tscircuit/core/pull/1617))
- Added Via-grid board support to KiCad→Circuit-JSON conversion ([kicad-to-circuit-json #20](https://github.com/tscircuit/kicad-to-circuit-json/pull/20))
- KiCad→Circuit-JSON converter now parses `gr_poly` and exports polygon SMT pads ([kicad-to-circuit-json #21](https://github.com/tscircuit/kicad-to-circuit-json/pull/21))
- Expanded KiCadTS schema with `GrCircle`, `GrRect`, `GrPoly`, `Group`, `graphicsPoly`, `graphicsRect` and string `fill` token ([kicadts #18](https://github.com/tscircuit/kicadts/pull/18), [#19](https://github.com/tscircuit/kicadts/pull/19), [#20](https://github.com/tscircuit/kicadts/pull/20), [#21](https://github.com/tscircuit/kicadts/pull/21))
- Runframe file list now sorts naturally (e.g., 2 < 10) ([runframe #1673](https://github.com/tscircuit/runframe/pull/1673))
- Error list in Runframe is now expandable, mirroring browser-console behavior ([runframe #1672](https://github.com/tscircuit/runframe/pull/1672))
- Fixed incorrect proxy URL in the import dialog on tscircuit.com ([tscircuit.com #1952](https://github.com/tscircuit/tscircuit.com/pull/1952))

# Changelog 2026-05

- Autorouter workflow overhauled
  • Rectangular reroute phases now supported [core #2249](https://github.com/tscircuit/core/pull/2249)
  • Components can be routed inside an autorouting phase [core #2236](https://github.com/tscircuit/core/pull/2236)
  • Phase regions accept an explicit shape field [props #659](https://github.com/tscircuit/props/pull/659) and tolerance settings [props #660](https://github.com/tscircuit/props/pull/660)
  • Reroute without phaseIndex automatically inserts a default phase [core #2260](https://github.com/tscircuit/core/pull/2260)
  • Platform-provided autorouter functions can now be injected [core #2256](https://github.com/tscircuit/core/pull/2256)
  • Users may pass custom DRC parameters to autoroutingphase [core #2267](https://github.com/tscircuit/core/pull/2267)

- New API to completely strip traces: `unrouteCircuitJson` [core #2245](https://github.com/tscircuit/core/pull/2245)

- DRC & clearance accuracy
  • Polygon-aware checks for rotated pads / plated holes [checks #143](https://github.com/tscircuit/checks/pull/143)
  • Via-to-pad clearance solver added to repair toolkit [high-density-repair03 #8](https://github.com/tscircuit/high-density-repair03/pull/8)
  • False “missing connection” errors fixed for multi-pad ports [checks #144](https://github.com/tscircuit/checks/pull/144)

- Plated-hole & footprint handling
  • Oval & pill plated-hole rotation preserved in JSON→TSCircuit [circuit-json-to-tscircuit #28](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/28) and JSON→KiCad [circuit-json-to-kicad #297](https://github.com/tscircuit/circuit-json-to-kicad/pull/297)
  • Top & bottom courtyard layers supported in footprint generation [circuit-json-to-tscircuit #30](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/30)
  • Inflated footprints now stay on their original layer [core #2251](https://github.com/tscircuit/core/pull/2251)

- KiCad interchange gains richer data
  • Edge.Cuts `gr_circle` graphics parsed [kicad-to-circuit-json #72](https://github.com/tscircuit/kicad-to-circuit-json/pull/72)
  • JLCPCB part-number footprint props preserved [kicad-to-circuit-json #71](https://github.com/tscircuit/kicad-to-circuit-json/pull/71)
  • Native KiCad traces kept as `pcb_trace.route` with via support & connectivity graph [kicad-to-circuit-json #80](https://github.com/tscircuit/kicad-to-circuit-json/pull/80), [#85](https://github.com/tscircuit/kicad-to-circuit-json/pull/85), [#86](https://github.com/tscircuit/kicad-to-circuit-json/pull/86)
  • Source-net metadata exported [kicad-to-circuit-json #82](https://github.com/tscircuit/kicad-to-circuit-json/pull/82)

- Circuit-JSON schema updates
  • New `through_pad` route type [circuit-json #579](https://github.com/tscircuit/circuit-json/pull/579)
  • Imported `pcb_trace` elements now survive inflation [core #2262](https://github.com/tscircuit/core/pull/2262)
  • Via objects may omit layer refs and use a `layers` array [props #668](https://github.com/tscircuit/props/pull/668)

- Gerber exporter can now emit 8-layer boards [circuit-json-to-gerber #85](https://github.com/tscircuit/circuit-json-to-gerber/pull/85)

- Schematic usability
  • Group-box pin placement honors port direction [core #2239](https://github.com/tscircuit/core/pull/2239)
  • Inner-label collision solver added [schematic-placement-analysis #18](https://github.com/tscircuit/circuit-json-schematic-placement-analysis/pull/18)

- UI polish: Order dialog CSS is now scoped to prevent host-page conflicts [order-dialog #1](https://github.com/tscircuit/order-dialog/pull/1)

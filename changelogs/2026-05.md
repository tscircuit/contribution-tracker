# Changelog 2026-05

- Autorouting phases: rectangular reroute regions and new `shape`/tolerance options, with default phase insertion and per-phase DRC params ([core #2249](https://github.com/tscircuit/core/pull/2249), [core #2260](https://github.com/tscircuit/core/pull/2260), [core #2267](https://github.com/tscircuit/core/pull/2267), [props #659](https://github.com/tscircuit/props/pull/659), [props #660](https://github.com/tscircuit/props/pull/660))
- New AutoroutingPhase `component` support enables component-driven routing steps ([core #2236](https://github.com/tscircuit/core/pull/2236))
- krt WASM autorouter improvements: preset added and correct nominal trace width used ([props #661](https://github.com/tscircuit/props/pull/661), [core #2297](https://github.com/tscircuit/core/pull/2297))
- `unrouteCircuitJson` helper for quickly stripping all routing from a board ([core #2245](https://github.com/tscircuit/core/pull/2245))
- Ports now more flexible: `name` and `direction` fields become optional ([props #658](https://github.com/tscircuit/props/pull/658), [props #657](https://github.com/tscircuit/props/pull/657))
- Expanded import support: vias preserved inside `pcb_trace`, connectivity graph built, copper pours parsed & exported ([kicad-to-circuit-json #86](https://github.com/tscircuit/kicad-to-circuit-json/pull/86), [kicad-to-circuit-json #85](https://github.com/tscircuit/kicad-to-circuit-json/pull/85), [circuit-json-to-kicad #264](https://github.com/tscircuit/circuit-json-to-kicad/pull/264))
- 8-layer Gerber generation now supported ([circuit-json-to-gerber #85](https://github.com/tscircuit/circuit-json-to-gerber/pull/85))
- Footprint generation upgrades: plated-hole rotation, top/bottom courtyard layers, fabrication-note paths ([circuit-json-to-tscircuit #28](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/28), [circuit-json-to-tscircuit #29](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/29), [circuit-json-to-tscircuit #30](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/30))
- DRC accuracy: polygon-aware pad checks, false missing-connection and mounting-hole/courtyard errors fixed ([checks #143](https://github.com/tscircuit/checks/pull/143), [checks #144](https://github.com/tscircuit/checks/pull/144), [checks #149](https://github.com/tscircuit/checks/pull/149))
- Visual output fixes: through-pad traces & knockout text now rendered correctly in SVG ([circuit-to-svg #560](https://github.com/tscircuit/circuit-to-svg/pull/560), [circuit-to-svg #561](https://github.com/tscircuit/circuit-to-svg/pull/561))
- STEP export size reductions with shared hole geometry, pill-vertex reuse and model de-duplication ([circuit-json-to-step #90](https://github.com/tscircuit/circuit-json-to-step/pull/90), [circuit-json-to-step #92](https://github.com/tscircuit/circuit-json-to-step/pull/92), [circuit-json-to-step #88](https://github.com/tscircuit/circuit-json-to-step/pull/88))
- Net/footprint UX: `nominalTraceWidth` added to Net objects and JLCPCB footprint autocomplete/library mapping introduced ([props #669](https://github.com/tscircuit/props/pull/669), [props #671](https://github.com/tscircuit/props/pull/671), [eval #2655](https://github.com/tscircuit/eval/pull/2655))
- Blind/rotated plated-hole and blind-via fabrication issues fixed for Gerber & KiCad outputs ([circuit-json-to-gerber #89](https://github.com/tscircuit/circuit-json-to-gerber/pull/89), [circuit-json-to-kicad #297](https://github.com/tscircuit/circuit-json-to-kicad/pull/297))

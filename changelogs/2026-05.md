# Changelog 2026-05

- Autorouting regions can now be defined by shape, enabling rectangular reroute phases and component phases ([core #2249](https://github.com/tscircuit/core/pull/2249), [core #2236](https://github.com/tscircuit/core/pull/2236), [props #659](https://github.com/tscircuit/props/pull/659))
- Ports are more flexible – `name` and `direction` are now optional ([props #658](https://github.com/tscircuit/props/pull/658), [props #657](https://github.com/tscircuit/props/pull/657))
- New `schematic` section added to Circuit JSON ([core #2234](https://github.com/tscircuit/core/pull/2234))
- New `unrouteCircuitJson` helper lets you strip routes from a design ([core #2245](https://github.com/tscircuit/core/pull/2245))
- Group schematic boxes use port‐direction to place pins correctly ([core #2239](https://github.com/tscircuit/core/pull/2239))
- Fixed incorrect placement of named ports in schematic boxes ([core #2237](https://github.com/tscircuit/core/pull/2237))
- Fixed subcircuit anchor positions in Circuit JSON ([core #2238](https://github.com/tscircuit/core/pull/2238))
- Inflated footprints now stay on their original layer when saved ([core #2251](https://github.com/tscircuit/core/pull/2251))
- Corrected clearance parameter mapping for via-to-pad checks ([core #2250](https://github.com/tscircuit/core/pull/2250))
- Polygon-aware DRC checks added for rotated pads & plated holes ([checks #143](https://github.com/tscircuit/checks/pull/143))
- Gerber export now supports 8-layer boards ([circuit-json-to-gerber #85](https://github.com/tscircuit/circuit-json-to-gerber/pull/85))
- KiCad `.kicad_pcb` files can be imported directly as Board components ([eval #2574](https://github.com/tscircuit/eval/pull/2574))
- Circuit-to-TSCircuit converter:
  • Plated hole rotation supported for oval & pill holes ([#28](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/28))
  • Fabrication note paths rendered on the fab layer ([#29](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/29))
  • Courtyard generation for top & bottom layers ([#30](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/30))
- Autorouter preserves obstacle geometry & rotation when round-tripping Circuit JSON ([autorouter #1125](https://github.com/tscircuit/tscircuit-autorouter/pull/1125))
- Added via-to-pad clearance solver for tighter designs ([autorouter #1113](https://github.com/tscircuit/tscircuit-autorouter/pull/1113))
- KiCadTS now round-trips the `pic_programmer` example and parses the full Arduino repo ([kicadts #37](https://github.com/tscircuit/kicadts/pull/37), [kicadts #35](https://github.com/tscircuit/kicadts/pull/35))
- Edge.Cuts `gr_circle` is now preserved when converting KiCad → Circuit JSON ([kicad-to-circuit-json #72](https://github.com/tscircuit/kicad-to-circuit-json/pull/72))
- Footprint property `JLCPCB Part` is retained during KiCad import ([kicad-to-circuit-json #71](https://github.com/tscircuit/kicad-to-circuit-json/pull/71))
- Order dialog CSS is now scoped to avoid breaking host page styles ([order-dialog #1](https://github.com/tscircuit/order-dialog/pull/1))

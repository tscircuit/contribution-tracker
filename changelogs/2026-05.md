# Changelog 2026-05

- Autorouter
  - Rectangular reroute-phase regions & component support added ([core#2249](https://github.com/tscircuit/core/pull/2249), [core#2236](https://github.com/tscircuit/core/pull/2236))
  - Autorouting-phase regions can now carry a shape property ([props#659](https://github.com/tscircuit/props/pull/659))
  - KRT WASM preset & default trace-width handling introduced ([props#661](https://github.com/tscircuit/props/pull/661), [core#2297](https://github.com/tscircuit/core/pull/2297))
  - Region reroute works without phaseIndex and preserves imported pcb_trace elements ([core#2260](https://github.com/tscircuit/core/pull/2260), [core#2262](https://github.com/tscircuit/core/pull/2262))
  - Reroute reachability fixes for overlapping targets & same-root nets ([autorouter#1173](https://github.com/tscircuit/tscircuit-autorouter/pull/1173), [autorouter#1177](https://github.com/tscircuit/tscircuit-autorouter/pull/1177))

- Design-rule checking / clearance
  - Polygon-aware DRC for rotated pads & plated holes ([checks#143](https://github.com/tscircuit/checks/pull/143))
  - Via-to-pad clearance solver added ([high-density-repair03#8](https://github.com/tscircuit/high-density-repair03/pull/8))
  - Mounting holes inside their own courtyard no longer flagged ([checks#149](https://github.com/tscircuit/checks/pull/149))

- PCB / fabrication output
  - Up to 8-layer Gerber export supported ([circuit-json-to-gerber#85](https://github.com/tscircuit/circuit-json-to-gerber/pull/85))
  - Copper-pour export for KiCad generation ([circuit-json-to-kicad#264](https://github.com/tscircuit/circuit-json-to-kicad/pull/264))
  - Blind-via drill output fixed ([circuit-json-to-gerber#89](https://github.com/tscircuit/circuit-json-to-gerber/pull/89))
  - Rotated pill plated-hole handling corrected across toolchain ([circuit-json-to-kicad#297](https://github.com/tscircuit/circuit-json-to-kicad/pull/297), [circuit-json-to-gerber#86](https://github.com/tscircuit/circuit-json-to-gerber/pull/86))
  - STEP export significantly smaller via shared hole geometry & vertex dedup ([circuit-json-to-step#90](https://github.com/tscircuit/circuit-json-to-step/pull/90), [circuit-json-to-step#92](https://github.com/tscircuit/circuit-json-to-step/pull/92))

- Circuit-JSON & schema
  - New through_pad route type introduced ([circuit-json#579](https://github.com/tscircuit/circuit-json/pull/579))
  - Via object now supports optional layer refs & layers array ([props#668](https://github.com/tscircuit/props/pull/668))

- KiCad / import pipeline
  - KiCad 10.0 support + Edge.Cuts circles & zone handling ([kicad-to-circuit-json#74](https://github.com/tscircuit/kicad-to-circuit-json/pull/74), [#72](https://github.com/tscircuit/kicad-to-circuit-json/pull/72), [#89](https://github.com/tscircuit/kicad-to-circuit-json/pull/89))
  - Knock-out board text preserved round-trip ([kicad-to-circuit-json#87](https://github.com/tscircuit/kicad-to-circuit-json/pull/87))

- Viewer & tooling UX
  - PCB-viewer “Bounds” tool for box-selection ([pcb-viewer#864](https://github.com/tscircuit/pcb-viewer/pull/864))
  - Runframe can autoroute a user-selected PCB region directly ([runframe#3415](https://github.com/tscircuit/runframe/pull/3415))
  - CLI supports static file loading & concurrent STEP generation ([cli#2987](https://github.com/tscircuit/cli/pull/2987), [cli#3019](https://github.com/tscircuit/cli/pull/3019))

- SVG / Canvas rendering
  - Through-pad PCB traces now render correctly in SVG & bounds calc ([circuit-to-svg#560](https://github.com/tscircuit/circuit-to-svg/pull/560))
  - Copper knockout text anchor alignment fixed in SVG & Canvas ([circuit-to-svg#561](https://github.com/tscircuit/circuit-to-svg/pull/561), [circuit-to-canvas#236](https://github.com/tscircuit/circuit-to-canvas/pull/236))

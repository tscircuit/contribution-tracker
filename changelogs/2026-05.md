# Changelog 2026-05

- Added rectangular reroute phases and component-scoped autorouting phases for finer re-routing control ([core #2249](https://github.com/tscircuit/core/pull/2249), [core #2236](https://github.com/tscircuit/core/pull/2236))
- Autorouter upgrades: new `krt` WASM preset, ability to pass a custom platform autorouter, default phase auto-insertion when `phaseIndex` is omitted ([props #661](https://github.com/tscircuit/props/pull/661), [core #2256](https://github.com/tscircuit/core/pull/2256), [core #2260](https://github.com/tscircuit/core/pull/2260))
- Autorouting-phase regions now support custom shapes and tolerance settings ([props #659](https://github.com/tscircuit/props/pull/659), [props #660](https://github.com/tscircuit/props/pull/660))
- Ports no longer require `name` or `direction`, and resistor footprint shorthands are now auto-mapped ([props #658](https://github.com/tscircuit/props/pull/658), [props #657](https://github.com/tscircuit/props/pull/657), [props #662](https://github.com/tscircuit/props/pull/662))
- New `unrouteCircuitJson` helper to strip existing routing from designs ([core #2245](https://github.com/tscircuit/core/pull/2245))
- Group schematic boxes: pins now placed by port direction, named arrangements fixed, and inner-label collision solver added ([core #2239](https://github.com/tscircuit/core/pull/2239), [core #2237](https://github.com/tscircuit/core/pull/2237), [schematic-placement-analysis #18](https://github.com/tscircuit/circuit-json-schematic-placement-analysis/pull/18))
- Footprint generation & export: plated-hole rotation support, courtyard (top/bottom & circle) layers, fabrication-note paths, and 8-layer Gerber output ([cj-to-ts #28](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/28), [cj-to-ts #30](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/30), [cj-to-ts #32](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/32), [cj-to-ts #29](https://github.com/tscircuit/circuit-json-to-tscircuit/pull/29), [cj-to-gerber #85](https://github.com/tscircuit/circuit-json-to-gerber/pull/85))
- KiCad import boosts: full KiCad 10.0 support, Edge.Cuts `gr_circle`, JLCPCB part-number retention, Arduino repo parsing, and pic_programmer round-trip ([k2cj #74](https://github.com/tscircuit/kicad-to-circuit-json/pull/74), [k2cj #72](https://github.com/tscircuit/kicad-to-circuit-json/pull/72), [k2cj #71](https://github.com/tscircuit/kicad-to-circuit-json/pull/71), [kicadts #35](https://github.com/tscircuit/kicadts/pull/35), [kicadts #37](https://github.com/tscircuit/kicadts/pull/37))
- `kicad_pcb` files can now be imported as Board components with exported `boardContentCircuitJson` ([eval #2574](https://github.com/tscircuit/eval/pull/2574), [eval #2595](https://github.com/tscircuit/eval/pull/2595))
- New polygon-aware DRC checks for rotated pads and plated holes ([checks #143](https://github.com/tscircuit/checks/pull/143))
- Via-to-pad clearance solver added to high-density repair workflow ([high-density-repair03 #8](https://github.com/tscircuit/high-density-repair03/pull/8))
- Order dialog CSS is now scoped to prevent host-page style clashes ([order-dialog #1](https://github.com/tscircuit/order-dialog/pull/1))
- Interactive viewer can calculate cell boundaries for layout analysis ([tscircuit #3144](https://github.com/tscircuit/tscircuit/pull/3144))
- Bug fixes: inflated footprints persist on correct layer, correct net-label behaviour, subcircuit anchor restored, 3D renderer no longer shows fallback boxes for empty models ([core #2251](https://github.com/tscircuit/core/pull/2251), [core #2255](https://github.com/tscircuit/core/pull/2255), [core #2238](https://github.com/tscircuit/core/pull/2238), [cj-to-gltf #160](https://github.com/tscircuit/circuit-json-to-gltf/pull/160))
- Added PCB snapshot drill-hole color flag for KiCad export ([cj-to-kicad #287](https://github.com/tscircuit/circuit-json-to-kicad/pull/287))
